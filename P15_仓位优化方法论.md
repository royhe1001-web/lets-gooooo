# 仓位优化方法论 — P15实盘经验总结

**日期**: 2026-05-14
**来源**: DualStream-CSI500 指数增强策略 P14→P15 优化历程
**适用范围**: 任何基于ML排名的动态仓位策略

---

## 一、核心原则

### 1. 牛市集中，熊市空仓（不要为了分散而分散）

| 错误做法 | 正确做法 |
|---------|---------|
| 牛市分散持仓降低风险 | 牛市集中2-4只最强股，单只敢上40-50% |
| 防御期强制买防守股 | 防御期允许空仓(0%配比)，留现金等牛市 |
| 所有市场状态下都要有仓位 | 中性50/50，防御0/0，踏空好过亏钱 |

**数据验证**: 防御仓位从72%降到52%后，牛熊仓位差距从-7pp反转到+12pp，超额+11.6pp。

### 2. 利润必须再投资（复利的关键）

```python
# ❌ 错误: 基于初始资金分配
allocator.capital = self.config.total_capital  # 永远200k

# ✅ 正确: 基于当前总权益分配
current_eq = self.cash + sum(position_value)
allocator.capital = max(current_eq, total_capital)
```

**数据验证**: 此单行改动贡献 +2.6pp 超额。不做的后果是利润累积为闲置现金（典型仓位只有60-70%）。

### 3. 买入上限基于初始资金而非剩余现金

```python
# ❌ 错误: 顺序买入时cash递减，后买股票被截断
target = min(target_amount, self.cash * 0.25)

# ✅ 正确: 基于初始资金，不受买入顺序影响
target = min(target_amount, self.config.total_capital * 0.25)
```

**数据验证**: 修复后仓位从56%跃升至66%，+9.3pp。典型场景：买第5只时剩余现金只剩20%，实际只投了5%仓位。

### 4. 整手数约束的代价

A股交易以100股为单位（1手），买入金额向下取整：

```python
n_shares = int(target / price / 100) * 100  # 整手向下取整
cost = n_shares * price                      # 实际投入 < 目标金额
```

- 高价股（>50元）浪费比例小（100股=5000元，取整损失<2500）
- 低价股（<10元）浪费比例大（100股=1000元，取整精度高）
- **多只小额分散买入时累积损失可达 10-15% 资金**

**对策**:
- 牛市集中时减少持仓数（2-4只），单只金额大 → 取整损失占比低
- 熊市分散时接受取整损失（安全第一）
- `min_single_amount` 门槛设为 8,000（太低无意义，整手取整后可能买到零股）

## 二、状态驱动的配比设计

### 底仓随牛市强度分级

```python
STATE_ALLOCATION = {
    AGGRESSIVE: (0.10, 0.90),   # 10%底仓, 90%纯Alpha
    BULLISH:    (0.25, 0.75),   # 25%底仓压阵, 75%重仓精选
    NEUTRAL:    (0.50, 0.50),   # 股债平衡
    DEFENSIVE:  (0.00, 0.00),   # 允许空仓, 不强制买任何股票
}
```

**设计原则**:
- 牛市越强底仓越少（让Alpha尽情发挥）
- 防御市允许配比为0（不强制买防守股）
- 底仓选大市值加权票，卫星选ML排名票

### 单票上限随状态动态调整

```python
if state in ('aggressive', 'bullish'):
    state_max_single = total_capital * 0.50  # 牛市中单只可达50%
else:
    state_max_single = total_capital * 0.20  # 熊市收紧到20%
```

### 状态切换行为

| 切换 | 行为 | 原因 |
|------|------|------|
| 升级（熊→牛） | 卖弱底仓 + 买强卫星 | 旧的防守股该让位给Alpha |
| 降级到防御 | 留最强卫星 + 不补仓 | 不要为分散而分散，留现金等机会 |
| 同级别切换 | 不强清仓 | 避免摩擦成本 |

**失败教训**: 
- ❌ 状态切换全清仓重建 → 摩擦成本吞掉超额（16.9%）
- ❌ 降级时强制补防守股 → 徒增分散不增收益
- ❌ 卖出后立即补仓 → 往往接盘高波动陷阱

## 三、Exit信号体系的经验

### 信号阈值经验值

| 参数 | 当前值 | 经验 |
|------|:---:|------|
| rank_sell_threshold | 8% | 牛市放宽到12%会反向（该卖不卖） |
| sell_score_threshold | 2 | 牛市提到3会拖死仓位 |
| min_holding_days | 5 | 盈利股额外保护+1分阈值 |
| trailing/drawdown | 3.8%(牛)~6.0%(防) | P7已验证有效，不动 |

**核心结论**: 现有的三信号投票体系（排名+砖型+止损）已经足够精准，不需要额外加排名轮换层。单独排名强制轮换全部反向（-13pp到-86pp）。

### 不要做的事

- ❌ 排名轮换（rank > 50% 强制卖）→ -13.4pp
- ❌ 缩短exit检查周期 → 无影响，状态切换已触发调仓
- ❌ 趋势过滤器（涨就不进防御）→ OAMV量价结合比简单价格趋势更准

## 四、调仓周期与执行价格

### 周期对齐

调仓周期与持有周期一致（T+4），但实际上状态切换才是调仓的主要触发源。周期天数在3-5天范围内对结果无影响。

### 执行价格

回测使用T日收盘价成交。改为T+1开盘价后超额从39.9%暴跌到2.9%。原因：
- 实盘中T+1开盘通常有0.5-1%滑点
- 累积80+次交易后滑点吃光超额
- **实盘对策**: 用限价单在开盘竞价执行，控制滑点在0.3%以内

实现新的仓位策略前，验证以下条目：

- [ ] allocator基于当前总权益还是初始资金？
- [ ] 买入上限基于total_capital还是self.cash？
- [ ] 防御/熊市是否允许空仓（配比=0,0）？
- [ ] 牛市单票上限是否足够大（≥40%）？
- [ ] 牛市持仓数是否足够集中（2-4只）？
- [ ] 状态切换时是否避免全清仓重建？
- [ ] 利润增长后是否自动再投资？
- [ ] 整手约束导致的闲置资金是否可接受？
- [ ] min_single_amount设置是否合理（≥8000）？
