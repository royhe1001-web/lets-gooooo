## 一、策略目标

基于超卖反弹的砖型反转形态，叠加 OAMV 市场状态感知和 P15 集中仓位管理，在 A 股主板捕捉 T+1 ~ T+4 短线反转机会。目标年化收益 ≥ 80%，最大回撤 < 30%，胜率 ≥ 40%。

## 三、选股体系

### 自定义技术指标（5 类）

所有指标从日线 OHLCV 计算：

| 类别 | 指标 | 作用 |
|------|------|------|
| KDJ | J 值 | 超卖判定 (入场要求前日 J<20) |
| MACD | MACD_HIST | 背离检测 (止盈信号) |
| 砖型图 | CC/CO/绿转红 | 趋势确认 + 双策略共振加权 |
| 白/黄线 | winner/loser 支撑阻力 | 收盘必须站上黄线 |
| 成交量 | 缩量/放量/均量比 | 抛压衰竭确认 |

### 入场条件（8 项 AND 逻辑）

```
① 前日 J < 20（超卖区域）
② 当日涨幅 > 动态阈值（agg=4% / norm=6% / def=禁止）
③ 当日 J < 动态阈值（agg=65 / norm=55）
④ 放量（volume > 前日）
⑤ 上影线 < 动态阈值（agg=4.0% / norm=3.0%）
⑥ 白线 > 黄线
⑦ 收盘 > 黄线
⑧ 收盘 > MA20（趋势过滤，排除下跌趋势假反弹）
```

**强势股放宽通道**: 20 日涨幅 > 20% 的强势股，③ 放宽到 J<70、⑤ 放宽到 4.5%，但权重 ×0.8。

### 权重因子（11 层叠加，上限 5x）

| # | 条件 | 乘数 | 含义 |
|---|------|:---:|------|
| 1 | 当日涨幅 > 9% | **×2.5** | 强动能加成 |
| 2 | 跳空高开 > 2% | ×1.5 | 资金抢筹确认 |
| 3 | 前日缩量 | ×1.2 | 抛压衰竭 |
| 4 | 深度缩量（<20日均量80%） | ×1.3 | 极致衰竭 |
| 5 | 前 10 日回调 > 5% | ×1.2 | 回调深度确认 |
| 6 | 上影线显著 | ×0.7 | 冲高回落折价 |
| 7 | 强势股放宽入场 | ×0.8 | 阈值放宽稀释 |
| 8 | Spring ∩ 砖型 CC 共振 | ×1.5 | 双策略确认 |
| 9 | ~~板块动量~~ (已关闭) | — | 加成仅 +1.1pp，已清理 |

## 四、OAMV 状态系统

### 三状态定义

| 状态 | OAMV 阈值 | 入场门槛 | 止损缓冲 | 开仓 |
|------|:---:|------|:---:|:---:|
| AGGRESSIVE | ≥ 3.0 | 涨幅 4%, J<65 | entry_low × 0.95 | 允许 |
| NORMAL | -2.0 ~ 3.0 | 涨幅 6%, J<55 | entry_low × 0.97 | 允许 |
| DEFENSIVE | ≤ -2.0 | 禁止入场 | entry_low × 0.99 | **禁止** |

| 参数 | def2.0 | agg2.5 | agg1.0 |
|------|:---:|:---:|:---:|

激进 OAMV 参数在 P15 集中持仓下不再单调递增 — **def2.0（保守）反而最优**。

## 五、投资策略（P15 仓位管理）

### 仓位规则

| 规则 | 值 | 来源 |
|------|:---:|------|
| 最大持仓数 | 4 只 | 牛市集中 |
| 最小持仓数 | 0 只 | 防御允许空仓 |
| 单票上限 | 当前总权益 × **50%** | 统一上限 |
| 最小买入额 | ¥8,000 | 整手约束门槛 |
| 每日最大买入 | 3 只 | 防止过度交易 |

### 仓位计算公式

```
total_eq  = cash + Σ(当前持仓市值)
capital   = max(total_eq, 初始资金)
slots     = max(1, 4 - 当前持仓数 + 1)
base      = capital / slots
wf        = min(signal_weight / 2.0, 2.5)
alloc     = min(base × wf, capital × 50%, cash × 98%)
shares    = int(alloc / price / 100) × 100    ← 整手取整
```

**关键**: 基于总权益（非剩余现金），利润自动再投资；买入上限不受买入顺序影响。

### 轮换逻辑

| 条件 | 操作 |
|------|------|
| 新信号权重 > 最弱持仓权重 + 0.3 | 替换 |
| 最弱持仓微利（>3%）+ 新信号更强 | 替换 |
| 最弱持仓亏损（<-2%）+ 新信号更强 | 止损替换 |
| 有空位（<4 只） | 直接买入 |

### 状态切换行为

| 切换 | 操作 |
|------|------|
| 升级（防御→普通→激进） | 卖出弱持仓 (权重<1.2 且微利<3%)，留现金给 Alpha |
| 降级到 DEFENSIVE | 只留最强 1 只（或全部清仓），不补仓 |
| 同级 | 不强制清仓，正常轮换 |

## 六、离场系统

| 退出 | 触发条件 | 说明 |

- 常规：T+4 交易日（或状态切换立即触发）
- 非调仓日：仅检查止损，不生成买入

## 八、关键技术决策

| 决策 | 结论 | 证据 |
|------|------|------|
| P15 集中持仓（总权益基准+单票50%） | **核心突破** | baseline 从 +19.6% → +64.9%（+45pp） |
| 利润再投资（total_eq 而非 initial_cap） | 复利引擎 | 独立贡献 +2.6pp |
| 防御期允许空仓（min_positions=0） | 风控关键 | 避免熊市强制持仓 |
| 整手取整 + min ¥8000 | 减少摩擦 | 多只小额分散累积损失 10-15% |
| 板块动量因子 | **已关闭** | 加成仅 +1.1pp，Stage 1 筛选淘汰 |
| 激进 OAMV 参数 | 不再单调 | P15 集中持仓下 def2.0 反而最优 |
| 训练窗口 Range 2016-2017 | 中性 | 有无该窗口 Test 2026 均为 +19.6% |
| 收盘价成交 | 正确 | T+1 开盘滑点吃掉超额（参考 CSI500） |
| 主板块（排除创业板/科创板） | 选股基础 | 市值分位 + 流动性双重过滤 |
| 动态 OAMV 阈值（入场门槛随状态调节） | 有效 | aggressive 降门槛捕捉更多机会 |

## 九、回测与训练

### 回测引擎

| 组件 | 说明 |
|------|------|
| 引擎 | OAMVSimEngine（P0-P4 内置） |
| 选股 | Spring B2 8 条件 AND + 11 层权重 |

| 市值过滤 | 月度百分位 + OAMV 状态感知 |
| 流动性过滤 | 20 日均成交额 > 2000 万 或 换手率 > 0.5% |
| 成交价 | T 日收盘价 |
| 手续费 | 买入 0.03%, 卖出 0.13%（含印花税） |

### 股票池

```
主板前缀: 000, 002, 003, 600, 601, 603, 605
排除: 创业板 (300/301), 科创板 (688/689)
预筛选: 市值分位 (aggressive=p70-p98) + 流动性
候选池: ~2100-3700 只 → OAMVSimEngine 精选 → 0-4 只持仓
```

### 训练方法

Spring B2 为**规则引擎**（非 ML 模型），参数通过 4 阶段多牛市网格搜索优化：

```
Stage 1: T+4 快速筛选 (31 参数集 × 4 牛市周期)
Stage 2: 全回测网格 (Top-5 × 全周期)
Stage 3: 验证 Val 2025H2
Stage 4: 测试 Test 2026
```

训练窗口：2014-2015, 2020-2021, 2024-2025H1（纯牛市）。Range 2016-2017 经测试为中性，已移除。
