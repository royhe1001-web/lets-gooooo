# BUG: Parquet 重复日期 + 复权价格不一致，1411只股票受影响

**发现**: 2026-05-15 Mac端回测验证  
**严重程度**: 致命 — 导致回测收益虚高至 +775%（真实收益 +32.6%）

---

## 现象

000070（特发信息）在 4/23 发生拆股（约10送10），parquet 文件中同一天存在两条记录：

```
2026-05-06 00:00:00  open=18.45  close=18.28   ← 未复权价
2026-05-06 15:00:00  open= 2.27  close= 2.27   ← 前复权价
2026-05-07 00:00:00  open=19.80  close=20.11   ← 未复权价
2026-05-07 15:00:00  open= 2.27  close= 2.23   ← 前复权价
```

引擎在 5/6 以 2.27（前复权）买入 34500 股，5/7 以 20.11（未复权）卖出，单票虚增 +786%。

## 影响范围

扫描 `ML_optimization/features/` 下所有 parquet 文件，**1411 只股票**存在重复日期（占总数 ~66%）。

## 根因分析

### 1. 日期时间分量不一致
同一天的数据带了不同时间戳（00:00:00 vs 15:00:00），导致：
- `drop_duplicates(subset='date')` 视两条为不同行，无法去重
- engine 的 `run()` 中 `all_trading_dates` set 也无法去重（pandas Timestamp 比较包含时间分量）

### 2. 复权处理不完整
k线更新脚本可能在发生股本变动时，新增了前复权价格行，但没有删除旧的未复权行，导致两种价格并存。

## Mac端临时修复

1. `phase2c_bull_grid_search.py` preload 时 `.dt.normalize()` 归一化后再去重
2. `phase2c_oamv_grid_search.py` run() 中 OAMV/signal 日期统一 normalize
3. `phase2c_oamv_grid_search.py` sell() 部分卖出整百取整

**但这些是治标不治本**，修复后回测收益从 +775% 骤降至 +32.6%，说明原先 P2 声称的 +116.6% 可能也包含虚假数据。

## Win端需修复（根因）

1. **k线更新脚本**：发生股本变动时，统一使用前复权（qfq）价格覆盖全历史，不要新增行
2. **日期统一归一化**：所有 parquet 的 date 列统一为 `YYYY-MM-DD` 格式（无时间分量）
3. **修复现有数据**：对已生成的 2125 个 parquet 文件批量清理，去重 + 统一前复权
4. **重新生成所有指标**：k线数据修复后，需要重新计算所有技术指标（`--recalc`）

---

**优先级**: Win端修复 k线管道 > 重算指标 > 重新推送 Mac端
